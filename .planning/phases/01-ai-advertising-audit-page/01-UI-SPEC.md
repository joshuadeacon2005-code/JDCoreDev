## 4. Typography

The codebase uses Tailwind's default type scale. The recurring pattern in services.tsx is small-set-of-sizes + italic-uppercase headings + tight tracking. Mirror it exactly.

| Role | Tailwind classes | Notes |
|------|------------------|-------|
| Section eyebrow | `text-accent font-black italic uppercase tracking-[0.3em] mb-4 text-sm block` (prefix `// `) | Mandatory above every section H2 |
| H1 (hero) | `text-5xl md:text-7xl font-black text-foreground tracking-tighter uppercase italic mb-6` | Use `<span className="text-primary">word</span>` for one accented word |
| H2 (section) | `text-4xl md:text-5xl font-black text-foreground tracking-tighter uppercase italic mb-4` (or `mb-6`) | Same accent-word pattern |
| H2 (final CTA) | `text-4xl md:text-6xl font-black tracking-tighter uppercase italic mb-6` | Final CTA uses larger md size |
| H3 (card title) | `text-xl md:text-2xl font-black uppercase italic tracking-tight` | Used inside benefit cards / pricing cards |
| H3 (small card) | `text-xl font-black uppercase italic tracking-tight` | Used in 2-col pricing-style cards |
| Card subtitle | `text-xs text-accent uppercase tracking-widest font-bold mb-4` | Used above tiered pricing card descriptions |
| Body lead | `text-lg text-muted-foreground max-w-2xl mx-auto leading-relaxed` | Sits below H1/H2 in hero/intro blocks |
| Body | `text-muted-foreground leading-relaxed` (sometimes `text-sm`) | Default paragraph text inside cards |
| Small body / list item | `text-sm text-muted-foreground leading-relaxed` | Card body copy |
| Bullet text | `text-xs uppercase tracking-wide` | Feature-list items inside benefit cards |
| Number prefix | `text-5xl font-black text-primary/15 italic leading-none` | Big "01" / "02" badge in benefit cards |

Line-heights are Tailwind defaults (`leading-relaxed` = 1.625, `leading-tight` = 1.25, `leading-none` = 1). Do not override.

**Two weights only:** `font-bold` (600) for emphasis, `font-black` (900) for headings/labels. Body inherits regular (400). This is what services.tsx uses; do not introduce `font-semibold`/`font-medium` outside the cases listed above.

## 5. Color

Color is locked to existing CSS vars. New page uses `text-primary`, `text-accent`, `text-muted-foreground`, `text-foreground`, `bg-background`, `bg-muted/20`, `bg-muted/30`, `border-border`, `border-border/50` — nothing else.

| Role | Token | HSL (light) | HSL (dark) | Where it's used |
|------|-------|-------------|------------|-----------------|
| Dominant (60%) | `--background` / `bg-background` | `160 15% 97%` | `200 20% 8%` | Page background, hero |
| Secondary (30%) | `--card` + `--muted` | `0 0% 100%` / `160 12% 94%` | `200 18% 11%` / `200 15% 15%` | `bg-muted/30` is the canonical "alt-row" surface for sections (used in services.tsx final CTA and Hosting & Maintenance right-column) |
| Accent (10%) | `--primary` AND `--accent` (same teal) | `165 55% 32%` | `165 55% 45%` | Both `text-primary` (in-headline word + icons + bullet dots + price) and `text-accent` (the `// eyebrow` label) — these resolve to the same teal token |
| Destructive | `--destructive` | `0 65% 55%` | `0 65% 55%` | NOT USED on this page (no destructive actions) |
| Body text | `--muted-foreground` | `200 10% 40%` | `160 10% 60%` | All paragraph text |
| Borders | `--border` | `160 10% 88%` | `200 15% 18%` | All bordered grids and dividers |

**Accent reserved for:**
1. The `// Section eyebrow` label (`text-accent`)
2. ONE accented word per heading (`<span className="text-primary">word</span>`)
3. Lucide icons inside cards (`text-primary` on a `h-5 w-5` icon)
4. Bullet dots (`bg-primary` on a `1.5h × 1.5w` rounded-full)
5. Pricing tier prices (`text-primary`)
6. The big translucent number badge in benefit cards (`text-primary/15`)
7. Primary CTA Button — handled by the `Button` component default

**Forbidden on this page:**
- `bg-primary` as a wash on whole sections (Button handles this internally; do not paint cards primary)
- Any new color, gradient, or RGBA literal
- Custom hex codes
- `text-destructive` (no destructive UI)

**Dark mode:** services.tsx is dark-mode aware via the existing CSS-var system. The new page inherits this automatically by using the same tokens. Do not write any `dark:` overrides; the tokens flip themselves.
