DATA MODEL CHANGES (ADD THESE TABLES/FIELDS)
A) Client enhancements
- Add to clients table:
  - address_line1 (nullable)
  - address_line2 (nullable)
  - city (nullable)
  - region (nullable)
  - postal_code (nullable)
  - country (nullable)
  - first_contact_date (nullable, date)

B) Project Process
- Create new table project_process_steps:
  - id
  - project_id (FK)
  - title (string)
  - description (text, nullable)
  - step_order (int)
  - status (planned | in_progress | done)
  - created_at

C) Prompt History (Project-specific)
- Create new table project_prompts:
  - id
  - project_id (FK)
  - prompt_title (string)
  - prompt_text (text)
  - output_summary (text, nullable)  // optional short summary; not required to store full outputs
  - tags (string, nullable)          // comma-separated tags OK for MVP
  - created_by_user_id (FK users)
  - created_at

D) Contracts / Agreements (Project-specific)
- Create new table project_agreements:
  - id
  - project_id (FK)
  - agreement_type (contract | sow | msa | nda | other)
  - title (string)
  - signed (bool)
  - effective_date (date, nullable)
  - expiry_date (date, nullable)
  - notes (text, nullable)
  - document_id (FK documents, nullable)  // link to stored PDF/doc upload when available
  - created_at

IMPORTANT: Use Alembic migrations for all schema changes.
