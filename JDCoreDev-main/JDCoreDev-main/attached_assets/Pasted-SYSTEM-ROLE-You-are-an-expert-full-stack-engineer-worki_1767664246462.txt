SYSTEM / ROLE
You are an expert full-stack engineer working inside the existing JD Core app in Replit. Add a “Hosting & Maintenance” box to every Project profile. The box must let the admin store and edit the hosting/maintenance agreement information (like the sample contract provided), and optionally render a formatted “Agreement Preview” for copy/paste. Keep UI minimal, dev-ish, and consistent with existing templates. Implement DB migrations, backend routes, templates, seed updates, and minimal tests. Do not change unrelated functionality.

PRIMARY OBJECTIVE
On each Project detail page (Admin + Client portal):
- Admin can enter and manage hosting & maintenance terms for that project
- Client can view a read-only summary once the project is completed
- Data should be structured (fields), not just a single blob of text
- Provide an optional “Agreement text” large field for full contract text

DISPLAY RULE
- Show the box ONLY when project.status == "completed" (finished).
- If not completed, show a small note: “Hosting & Maintenance becomes available once the project is completed.”

DATA MODEL (ALEMBIC MIGRATION REQUIRED)
Create table: project_hosting_terms (one row per project)
Fields:
- id (pk)
- project_id (FK projects, unique, indexed)
- start_date (date nullable)
- initial_term_months (int default 6)
- monthly_fee_cents (int nullable)
- currency (string default "USD")
- billed_in_advance (bool default true)
- invoice_due_days (int default 14)

Scope (store as text blocks, one per field, not huge):
- included_services (text nullable)   // bullet list style text
- excluded_services (text nullable)   // bullet list style text
- client_responsibilities (text nullable)
- availability_disclaimer (text nullable)
- term_extension_notes (text nullable) // e.g. extend 6 months or self-hosting option
- self_hosting_handover_notes (text nullable)
- ip_notes (text nullable)
- confidentiality_notes (text nullable)
- liability_cap_notes (text nullable)
- force_majeure_notes (text nullable)
- governing_law (string default "Hong Kong SAR")
- termination_notice_days (int default 30)

Agreement full text storage (optional):
- agreement_full_text (text nullable)

Metadata:
- status (enum: none|draft|active|ended) default draft
- updated_by_user_id (FK users nullable)
- updated_at (datetime)
- created_at (datetime)

OPTIONAL: link to signed document
- agreement_document_id (FK documents nullable)

BACKEND / SERVICES
- Add helper service: get_or_create_project_hosting_terms(project_id)
- Enforce permissions:
  - Admin can edit
  - Client can view only for their own client_id projects AND only if project.status=completed

Client portal:
- GET /portal/projects/{project_id} (extend existing page to show read-only hosting terms section)
