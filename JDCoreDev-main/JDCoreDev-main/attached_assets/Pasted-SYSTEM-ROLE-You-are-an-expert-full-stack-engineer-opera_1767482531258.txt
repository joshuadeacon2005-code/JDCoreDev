KEY WORKFLOWS (MUST IMPLEMENT)

A) Availability (Admin)
- Admin defines availability_rules
- Admin can block specific dates
- A date is available if:
  - weekday is allowed
  - date is not blocked
  - weekly and monthly capacity is not exceeded

B) Office Day Requests (Client)
- Client sees a 60-day calendar view showing available/unavailable dates
- Client may request one or more office days linked to a project
- Validate:
  - date is in the future
  - date is available
  - capacity rules are respected
  - project belongs to client
- Requests default to status = requested

C) Approval Flow (Admin)
- Admin can approve or reject office day requests
- Approval sets decided_by and decided_at
- Each action creates an activity_event

D) Payments & Milestones
- Admin creates milestones per project
- Project page shows:
  - total_value = sum(milestones.amount)
  - paid = sum(amount where status = paid)
  - outstanding = total_value - paid
- Client view is read-only

E) Documents
- Upload and download documents with role enforcement
- Store files at:
  /data/uploads/<client_id>/<project_id or _client>/<document_id>_<version>_<filename>
- Uploading a new version increments version number
- Each upload creates an activity_event

F) Activity Timeline
- Automatically log events for:
  - Lead creation
  - Office day request lifecycle
  - Milestone creation and status changes
  - Document uploads
  - Project status changes
